## 開発フロー

新しい機能を追加する際の推奨手順：

1. **Schemas層の定義**
   - Zodスキーマの作成
   - 型定義（リクエスト/レスポンス）

2. **Repositories層の実装**
   - データアクセスロジックの実装
   - Prismaクエリの最適化

3. **Services層の構築**
   - ビジネスロジックの実装
   - ファクトリー関数パターンの使用
   - バリデーションとエラーハンドリング

4. **Controllers層の作成**
   - リクエスト/レスポンス処理の実装
   - サービス層の呼び出し

5. **Routes層の作成**
   - OpenAPIルート定義
   - コントローラーとの紐付け

6. **依存性の組み立て**
   - `index.ts`で依存関係を構築
   - ルートの登録

7. **テストの作成**
   - ユニットテスト（各層）
   - 統合テスト（E2E）

詳細は各ガイドを参照してください。

## ディレクトリ構造

```
backend/src/
├── routes/              # OpenAPIルート定義
├── controllers/         # リクエスト/レスポンス処理
├── services/            # ビジネスロジック
├── repositories/        # データアクセス
├── schemas/             # スキーマと型定義
└── lib/                 # ユーティリティライブラリ
```
