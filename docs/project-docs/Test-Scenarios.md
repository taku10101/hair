## テスト戦略

### テスト方針

詳細なテスト戦略は以下のドキュメントを参照してください：

**テスト戦略ガイド**: [.ai-guide/testing.md](https://github.com/TLPropStation/web_template/blob/main/.ai-guide/testing.md)

### テストの種類

1. **ユニットテスト**: 個別の関数・メソッドの動作検証（カバレッジ目標: 80%以上）
2. **統合テスト**: データベースとビジネスロジックの統合検証
3. **APIテスト**: HTTPエンドポイントの動作検証
4. **コンポーネントテスト**: Reactコンポーネントのレンダリングと相互作用検証
5. **E2Eテスト**: エンドツーエンドのユーザーシナリオ検証

### テストツール

- **バックエンド**: Vitest、Hono testClient
- **フロントエンド**: Vitest、React Testing Library、Playwright
- **カバレッジ**: @vitest/coverage-v8

## 認証機能のテスト

### 1. ユーザー登録（POST /auth/signup）

#### 正常系

| テストケース | 入力 | 期待される結果 | 優先度 |
|------------|------|--------------|--------|
| 新規ユーザー登録成功 | email, password, name | 201 Created、ユーザー情報が返却される | 高 |

| パスワードが短すぎる | password = "1234567" (7文字) | 400 Bad Request、バリデーションエラー | 高 |
| 無効なメールアドレス形式 | email = "invalid-email" | 400 Bad Request、バリデーションエラー | 高 |
| 必須フィールドが欠けている | emailなし | 400 Bad Request、バリデーションエラー | 高 |
| 名前が空文字列 | name = "" | 400 Bad Request、バリデーションエラー | 中 |

## E2Eテスト

### ユーザー登録からログインまでの流れ

1. トップページにアクセス
2. 「新規登録」ボタンをクリック
3. メールアドレス、パスワード、名前を入力
4. 「登録」ボタンをクリック
5. ダッシュボードにリダイレクトされる
6. ログアウト
7. ログインページにアクセス
8. 登録したメールアドレスとパスワードを入力
9. 「ログイン」ボタンをクリック
10. ダッシュボードにリダイレクトされる

### ファイルアップロードとダウンロードの流れ

1. ログイン済みの状態でファイルページにアクセス
2. 「アップロード」ボタンをクリック
3. ファイルを選択
4. アップロードが完了し、ファイル一覧に表示される
5. アップロードしたファイルをクリック
6. ファイルがダウンロードされる

### 管理者によるユーザー管理の流れ

1. 管理者アカウントでログイン
2. ユーザー管理ページにアクセス
3. 「新規ユーザー招待」ボタンをクリック
4. メールアドレス、名前、パスワード、ロールを入力
5. 「招待」ボタンをクリック
6. ユーザー一覧に新規ユーザーが表示される
7. ユーザーの編集ボタンをクリック
8. ロールを変更して「保存」ボタンをクリック
9. ユーザー情報が更新される

## テスト自動化

### CI/CD パイプライン

GitHub Actions を使用してテストを自動実行します。

```yaml
# .github/workflows/test.yml
name: Test

on: [pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - uses: actions/setup-node@v4

      - name: Install dependencies
        run: pnpm install

      - name: Lint
        run: pnpm lint

      - name: Type check
        run: pnpm typecheck

      - name: Test with coverage
        run: pnpm test:coverage

      - name: Upload coverage
        uses: codecov/codecov-action@v3
```

```typescript
// backend/src/lib/testSetup.ts
beforeEach(async () => {

    prisma.user.deleteMany(),

  ])
})
```
